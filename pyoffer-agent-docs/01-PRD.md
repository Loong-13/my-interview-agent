# 01. 产品需求文档

## 1. 产品定位

PyOffer Agent 是一个面向 Python 后端与 AI Agent 应用开发方向应届生的 AI 面试准备工具。

它不是通用聊天机器人，而是围绕“某个候选人 + 某份简历 + 某个目标岗位”提供定制化训练。

## 2. 目标用户

### 主要用户

- Python 后端开发实习生
- Python 后端校招生
- LLM 应用开发实习生
- AI Agent 应用开发实习生

### 用户典型问题

- 简历写了项目，但不知道面试官会怎么追问
- Python / FastAPI / 数据库基础不够系统
- Agent / RAG 项目只会做 demo，不知道怎么讲工程细节
- JD 看得懂，但不知道自己和岗位差距在哪里
- 面试回答空泛，缺少结构和技术深度

## 3. 核心价值

系统需要帮助用户完成：

1. 判断简历是否匹配目标岗位
2. 找出简历中缺少的 Python 后端和 Agent 工程能力
3. 根据 JD 和简历生成个性化面试题
4. 模拟真实面试追问
5. 输出可执行的复盘报告
6. 生成下一轮训练建议

## 4. MVP 范围

### 必须实现

- 用户注册 / 登录
- 创建求职项目
- 上传简历
- 简历文本提取
- 粘贴岗位 JD
- 简历分析
- JD 分析
- 简历与 JD 匹配分析
- 面试题生成
- 文本模拟面试
- 面试报告生成

### 暂不实现

- 实时语音
- 录音回放
- 支付订阅
- 企业招聘端
- 社区题库
- 多人协作
- 浏览器插件

## 5. 核心用户流程

```text
注册 / 登录
  ↓
创建求职项目
  ↓
选择方向：Python 后端 / Agent 工程 / LLM 应用
  ↓
上传简历
  ↓
粘贴岗位 JD
  ↓
生成简历分析与匹配报告
  ↓
生成面试题
  ↓
开始文本模拟面试
  ↓
AI 追问
  ↓
结束面试
  ↓
生成面试复盘报告
```

## 6. 页面设计

### 6.1 登录注册页

功能：

- 邮箱注册
- 邮箱登录
- JWT 鉴权
- 登录态保持

### 6.2 项目列表页

展示用户创建的求职项目。

字段：

- 项目名称
- 目标公司
- 目标岗位
- 方向
- 最近一次匹配分
- 最近面试时间

操作：

- 创建项目
- 进入项目详情
- 归档项目

### 6.3 项目详情页

展示：

- 简历状态
- JD 状态
- 匹配分数
- 推荐练习方向
- 最近面试记录
- 下一步建议

核心按钮：

- 上传 / 替换简历
- 添加 / 更新 JD
- 生成匹配报告
- 开始模拟面试

### 6.4 简历与 JD 分析页

功能：

- 上传 PDF / DOCX 简历
- 展示提取文本
- 粘贴岗位 JD
- 触发 AI 分析
- 展示技能匹配、缺失能力、简历修改建议

### 6.5 模拟面试页

功能：

- 选择面试模式
- 展示 AI 问题
- 用户文本回答
- AI 给出简短反馈
- AI 继续追问
- 用户手动结束面试
- 自动生成报告

## 8. 核心指标

- 幻觉率

## 9. 主要风险

### 9.1 AI 输出泛泛而谈

解决方式：

- Prompt 中强制引用简历和 JD 证据
- 输出结构化 JSON
- 每个建议必须对应具体简历片段或面试回答

### 9.2 面试问题不够像真实面试

解决方式：

- 题目模板按方向维护
- 项目深挖问题优先来自简历项目
- 追问必须基于用户上一轮回答

### 9.3 文档解析不稳定

解决方式：

- MVP 优先支持 PDF / DOCX
- 允许用户手动编辑解析文本
- 保留 raw_text 供用户检查
